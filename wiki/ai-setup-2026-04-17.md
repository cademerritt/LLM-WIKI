# AI Setup — April 17, 2026

## Goal
All AI config lives on Drive E, shared between Linux and Windows. Switch OS and pick up exactly where you left off.

## What Was Built

### Drive E Structure
- `E:/claude-config/CLAUDE.md` — master profile and rules
- `E:/claude-config/memory/` — all memory files and session files
- `E:/claude-config/BOOT.md` — boot command text
- `E:/claude-config/boot-claude.sh` — Linux boot script
- `E:/claude-config/boot-claude.bat` — Windows boot script

### Linux Symlinks
- `/home/cade/.claude/CLAUDE.md` → `E:/claude-config/CLAUDE.md`
- `/home/cade/.claude/projects/-home-cade/memory/` → `E:/claude-config/memory/`

### Auto-Mount (fstab)
Drive E (UUID: `3ECC-0B35`, vfat) added to `/etc/fstab` — mounts automatically at `/media/cade/E` on every Linux boot.

### GitHub Backup
All files backed up to `cademerritt/claude-config` on GitHub.

## Boot Command (Linux)
Click **Bootup** desktop shortcut → copies full profile + 2 recent sessions to clipboard → paste into any Claude.

## Boot Command (Windows)
Double-click `E:\claude-config\boot-claude.bat` → same result.

## Hardware
- Drive E is partition `nvme1n1p2` (2.2G, vfat, labeled "E")
- Same physical drive as Linux (`nvme1n1`)
- NOT a separate SSD — a partition on the Linux NVMe drive
